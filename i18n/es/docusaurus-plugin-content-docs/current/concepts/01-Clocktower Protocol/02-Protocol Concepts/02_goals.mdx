## Objetivos

### Descentralizado

Los facilitadores de pago centralizados traen consigo una miríada de problemas, desde altas tarifas hasta el potencial de censura. Al utilizar un contrato blockchain descentralizado, no hay punto de falla, no hay censura y no hay control arbitrario de acceso.

### Inmutable

Un sistema que no puede ser cambiado es un sistema que no puede ser censurado.

### Fácil de Usar

En el pasado, configurar tu propio servicio de suscripción ha sido demasiado difícil para usuarios normales. Pero con Clocktower, si puedes usar una aplicación descentralizada, puedes crear o unirte a una suscripción.

### Económico

Las redes de pago tradicionales justifican cobrar altas tarifas debido a los altos costos de configurar una red tradicional. Pero al construir sobre la columna vertebral existente de blockchains compatibles con EVM, creemos que eventualmente podemos reducir el precio de las plataformas tradicionales.

### Sin Oráculos

Un oráculo es una fuente de datos de terceros. Las fuentes externas de información pueden ser incorrectas o manipuladas. Esto es demasiado riesgo para una plataforma de pago.

### Mínimo de tokens en el contrato

Los hackers hackean donde está el dinero. Los contratos tradicionales se han convertido en objetivos en gran parte porque dependen del modelo de "bóveda" donde todo el valor se almacena dentro del contrato. Buscamos dar vuelta este modelo buscando mantener el menor valor posible en el contrato. Esto hace que el contrato sea menos objetivo y permite a los usuarios asegurar sus propios fondos en sus propias billeteras.

### No se necesita token del protocolo

Creemos que un protocolo nunca debería necesitar su propio token para funcionar. Un token necesario para la funcionalidad crea fricción para el usuario cuando tiene que convertirlo y puede llevar a tokenomics inflacionarios. Si Clocktower alguna vez emitiera su propio token, sería únicamente para propósitos de gobernanza.
